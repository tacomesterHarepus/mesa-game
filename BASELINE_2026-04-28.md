## Key insights from this cleanup pass

1. **`overflow: hidden` clipping** — PlayerTurn's left column clips the staging zone text ("STAGING — N / M") because the 120px card buttons push it below the 154px content height. Any assertion using `toBeVisible()` on staging zone text will fail even when staging works correctly. The reliable visible proxy is the "END TURN" / "STAGE N MORE" button label change in the action-buttons column (unclipped).

2. **"STAGE N MORE" appears before `hasDiscarded=true`** — `endTurnBlocked` is calculated from `stagingNeeded` which doesn't gate on `hasDiscarded`. So "STAGE 1 MORE" is visible even while the discard edge function is in flight. Tests that wait for this label as a "discard done" signal will proceed while still in discard mode. The correct wait is for "DISCARD DONE" button label.

3. **`virus_pull` phase added in Phase 11** — Secret-actions and other tests that allowlist expected game phases needed `"virus_pull"` added. Three separate locations in secret-actions.spec.ts (two single-line, one multi-line format).

4. **Card button accessible names are UPPERCASE** — `CardStackGroup` renders card names as `(cardDef.name).toUpperCase()` inside the button. `getByRole("button", { name: "Pipeline Breakdown" })` still matches because Playwright does case-insensitive substring matching. Remove `exact: true` if used.
